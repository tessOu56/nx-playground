# 多領域實踐與系統整合深化 - 2024

## 🌐 上半年：接案與駐點專案經驗

2024 年上半年，我持續參與前公司專案的後續模組開發。

在專案變動後基於職涯規劃選擇離職，隨後以**自由接案與短期駐點**為主要工作模式，承接多項以 **React 技術**為核心的實務專案。

### 技術棧

React 18、TypeScript、Vite、Tailwind CSS、Zustand、WebSocket、SSE、Lottie.js

### 承接項目

#### 1. 專業形象官網開發

使用 **React + Vite** 建構模組化內容系統與 RWD 響應式版面。

**關鍵功能**：
- 結合 **Lottie.js** 實現動態過場動畫與互動效果
- 針對首次載入與動畫渲染進行效能優化（**Lazy Loading + Intersection Observer**）
- 專案部署依發案方策略進行整合
- 由我負責打包與環境設定，確保構建成果能無縫佈署於客戶伺服器

#### 2. 硬體整合介面開發

與後端與硬體工程師協作，採用：
- **WebSocket (WS)** 建立雙向即時通訊
- **Server-Sent Events (SSE)** 處理設備狀態推送

**實作內容**：
- 實現瀏覽器端即時顯示機器運轉資料
- 異常警示與回饋控制
- 設計**錯誤重試與資料緩衝（Replay Buffer）** 機制
- 讓系統在連線中斷時能自動恢復資料同步

**專案管理**：這些專案皆由我**獨立負責產品規劃、架構設計與進度排程**，在與設計師與後端協作間維持開發節奏與技術一致性。

---

## 技術深化與 IoT 資料流整合

上半年最重要的技術突破是**前端與物聯網（IoT）資料流整合**。

我針對高頻即時資料的傳輸與顯示設計出**多層資料流架構**：

### 雙通道即時架構

- **WS** 處理控制回應
- **SSE** 處理持續監測資料

### 資料緩衝與分層更新

- 以 **Zustand** 建立快取區
- 批次更新以降低 re-render

### 容錯與重連機制

- 採 **Exponential Backoff** 自動重連
- UI 層提示使用者狀態

### 可視化資料節流

- 針對高頻感測值以採樣率顯示
- 平衡效能與即時性

**這些經驗**讓我熟悉前端在 IoT 系統中處理即時資料流、流控與視覺化呈現的設計模式，也強化了跨層資料一致性與瀏覽器效能優化的能力。

## 下半年：加入數位產品公司

2024 年下半年，我加入一家**數位產品公司**，擔任**前端工程師兼系統分析（SA）**。

該公司主要產品為多模組數位平台，我加入時專案已開發一年。

### 核心任務

- ✅ 將產品團隊的需求文件轉化為**具體技術規格**
- ✅ 協調前後端 **API 與資料結構定義**
- ✅ 參與前端模組開發與 **Design System** 建置
- ✅ 協助外包與內部團隊溝通技術細節與開發優先順序

---

## 技術與系統實務

此階段專案架構為 **RESTful API**，後端以 **Go** 開發。

### 我的工作

我負責：
- 前端與 API 串接設計
- 錯誤回應與資料一致性管理
- 主要使用 **React + TypeScript** 撰寫業務模組
- 導入狀態管理與快取策略
- 確保多模組間資料傳遞流暢

## SSO（Single Sign-On）系統開發

因應公司多產品線的用戶身份整合需求，我與後端主管共同開發 **SSO（Single Sign-On）系統**。

### 實作方式

**核心機制**：Cookies 為核心認證機制

**整合內容**：
- 登入、登出與權限驗證流程
- 設計可重用的前端 **Auth Context** 模組

**成功成果**：
- ✅ 不同網域的產品維持一致登入狀態
- ✅ 改善使用者體驗
- ✅ 簡化維運程序

---

## Design System 建置

我與 **UI/UX 團隊**共同建立公司內部 **Design System**。

### 我的職責

- 將設計元件抽象化為可重用的 **React Component Library**
- 結合 **Storybook、Tailwind CSS 與 TypeScript Props Validation**
- 確保一致的互動行為、樣式與文件化展示

**影響**：
- ✅ 不同產品線能共用統一的設計語言
- ✅ 大幅提升前端開發效率與一致性

## 團隊協作與流程貢獻

我負責將**產品團隊的需求轉換為可執行的技術規格**。

### Spec Tracker 系統

使用 **Notion** 建立「**Spec Tracker（需求對應系統表）**」，集中管理：
- 設計稿
- API 對應
- 前端狀態
- 版本進度

**效益**：
- ✅ 外包團隊與內部工程師能以統一結構追蹤需求落實
- ✅ 能即時回溯版本變更
- ✅ 使產品決策與開發流程更透明、可持續迭代
