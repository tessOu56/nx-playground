# 架構重構與開發文化推進 - 2025

## 🧩 專案背景與重構契機

2025 年，我持續任職於數位產品公司，負責主要數位平台的**前端開發與系統架構**。

由於公司在年度營運目標下持續拓展業務，產品線快速增加多個新模組與功能，現有系統在**效能、維護與開發流程**上逐漸出現負擔。

為應對這波擴充，我**主導了前端專案的整體重構**，並導入 **Nx Monorepo 架構**以支撐多模組長期開發。

---

## 重構的主要目標與成果

### Nx Monorepo 導入

- 以 **Nx** 管理多應用與共用模組
- 建立跨專案可共享的邏輯與樣式庫

### 資料流優化

- 結合 **React Context + Zustand** 的層級式狀態管理
- 明確區分 Session、快取與業務邏輯
- 提升維護性與效能

### 效能提升

- 重新設計 **Lazy Loading、SSR 資料預取與 Code Splitting** 機制
- 降低載入時間並改善互動流暢度

### Design System 維護

- 持續與設計團隊協作
- 維護基於 **Storybook + Tailwind CSS + TypeScript** 的元件系統
- 確保所有模組維持一致的設計語言與文件化規範

---

## 下半年：前後台分離架構

下半年因產品方向調整，平台新增：
- **前後台分離架構**
- **內部管理系統**

### 我的職責

- 共用模組規劃
- 使用者角色權限設計
- 讓前後台能共用核心邏輯並維持獨立開發與部署彈性

---

## 技術深化與系統整合

2025 年的技術重心在於**鞏固前端與後端整合流程**，並建立**長期可維運架構**。

我以 **Next.js（App Router）** 為基礎，結合：
- TypeScript
- Vite
- Zustand
- RESTful API
- Go 後端協作

建立穩定技術體系。

### 型別與一致性控制

- 以 **TypeScript 嚴格模式**統一介面定義
- 為前後端 API 制定共用 **Response Interface**
- 減少資料錯誤

### 程式碼品質與維運流程

**CI 流程整合**：
- ESLint / Prettier / Type Check / Build Cache 自動化檢查
- 設計自動測試與版本標記腳本
- 確保每次合併均通過驗證

**部署與容器化**：
- 撰寫 **Dockerfile** 與多階段建置腳本
- 使用 **docker-compose** 進行整合測試
- 參與 **GitHub Actions** 流程設計
- 負責自動化建置、版本標記與環境部署（Staging / Production）

**透過這些實務**，我能掌握從開發、測試到上線的完整技術鏈，確保專案在多人協作與高頻部署下仍具穩定性與可追蹤性。

---

## 開發流程與團隊文化導入

隨著產品模組與人員增加，我推動團隊建立 **Scrum 開發文化**與**軟體開發制度化流程**。

### 主要推行項目

**Scrum 流程**：
- 與 PM 協同規劃 Sprint 流程
- 定期舉辦 Planning / Review / Retrospective

**任務管理**：
- 使用 **Notion + GitHub Projects** 管理開發任務與需求文件

**新成員指導**：
- 指導跨部門新進成員（行銷、PM、設計、前端實習生）
- 理解敏捷開發節奏與任務拆解方式

**Developer Handbook**：
- 制定統一分支策略
- Commit 標準
- 版本命名規範

**Code Review & Pair Programming**：
- 帶領新進工程師進行 Code Review
- Pair Programming 強化代碼品質與團隊默契

**這些制度**讓團隊形成共同的開發語言與節奏，提升跨部門溝通效率，也讓產品開發能在穩定節奏中持續推進。

我同時參與產品系統層的設計與維運：

- 定義前後台使用者層級
- 資料訪問控制設計

### API 規格與錯誤處理

- 建立統一 **Response Template** 與狀態碼標準

### Design System 長期維護

- 導入版本管理與 **Changelog** 機制
- 維持跨模組兼容性

### CI/CD 自動化流程

- 設計自動**健康檢查（Health Check）** 與 **Smoke Test** 機制
- 監控部署品質

**這些實作**讓我能以**架構視角**看待前端工程，確保產品在功能擴展下仍維持穩定性與擴充性。
